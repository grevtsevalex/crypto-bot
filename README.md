# Crypto Stoch RSI Bot

Telegram-бот по криптопарам Bybit: считает **Stochastic RSI на таймфрейме 1h** и отправляет уведомление подписчикам **только когда Stoch RSI = 100** (перекупленность). Параметры расчёта зафиксированы и совпадают с общепринятой формулой (в т.ч. как на Bybit/TradingView).

## Как это устроено

1. **Таймфрейм 1h** — используются часовые свечи Bybit (interval 60).
2. **Расчёт Stoch RSI** — RSI по Уайлдеру (период 14), затем стохастик по окну 14: `(RSI − min) / (max − min) × 100`.
3. **Сигнал** — при Stoch RSI ≥ порога (по умолчанию 99.99); лимит уведомлений за цикл (по умолчанию 10).
4. **Подписчики** хранятся в `subscribers.json`.

## Требования

- Go 1.24+
- Токен бота от [@BotFather](https://t.me/BotFather)

## Установка и запуск

```bash
cd crypto-bot
go build -o crypto-bot .

# Указать telegram_token в config.json (см. ниже)
./crypto-bot
```

При первом запуске без `config.json` создаётся файл с полями по умолчанию. Нужно заполнить `telegram_token`.

## Конфигурация

Поля в `config.json` (дефолты как в `config.example.json`):

| Параметр                 | Описание                          | По умолчанию |
|--------------------------|-----------------------------------|--------------|
| `telegram_token`         | Токен бота                        | —            |
| `subscribers_file`       | Файл подписчиков                  | `subscribers.json` |
| `period`                 | Период RSI/Stoch (7, 14, 21)      | 14           |
| `stoch_rsi_threshold`    | Сигнал при Stoch RSI ≥ значения   | 99.99        |
| `max_signals_per_cycle`  | Макс. уведомлений за проход       | 10           |
| `candle_limit`           | Число часовых свечей              | 100          |

Те же параметры можно менять в боте: **/settings** или кнопка «Настройки» (таймфрейм 1h не настраивается).

## Команды бота

| Команда     | Действие         |
|-------------|------------------|
| `/start`    | Главное меню     |
| `/settings` | Настройки        |
| `/status`   | Статус подписки  |
| `/stop`     | Отписаться       |
| `/help`     | Справка          |

## Параметры расчёта (зашиты в коде)

- **Таймфрейм:** 1h (60 мин)
- **Свечей:** 100
- **RSI период:** 14 (сглаживание Уайлдера)
- **Stoch период:** 14 (окно min/max RSI)
- **Условие сигнала:** Stoch RSI ≥ 99.5 (считается как 100)

Формула RSI совпадает с классикой (Bybit Learn, TradingView): первый RSI по SMA за 14 баров, далее сглаживание Уайлдера. Stoch RSI = (текущий RSI − min за 14) / (max − min) × 100.

## Структура проекта

```
crypto-bot/
├── main.go                 # Точка входа, цикл анализа (1h, Stoch RSI = 100)
├── config.json
├── config.example.json
├── subscribers.json
└── internal/
    ├── config/             # Только telegram_token и subscribers_file
    ├── exchange/           # Список пар и свечи Bybit
    ├── handlers/           # Подписка, отписка, статус, справка
    ├── notify/             # Рассылка при Stoch RSI = 100
    └── rsi/                # RSI по Уайлдеру + Stoch RSI
```

Не передавайте `telegram_token` в публичные репозитории; при необходимости добавьте `config.json` в `.gitignore`.
